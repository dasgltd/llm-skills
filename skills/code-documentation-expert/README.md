# Code Documentation Expert (LLM Skill)

> [!TIP]
> **LLM Skills** are customized instructions for AI Agents (like Google Antigravity, Claude Code, GitHub Copilot, Codex, etc.) to help them navigate complex, specialized workflows with ease. 

This specific skill transforms an AI Agent into a meticulous **Technical Writer and Documentation Expert**. It enforces rigorous standards for commenting, typing, and explaining codebases so they can be easily understood by other developers and future AI agents.

## The Problem
When asked to "document this code", AI agents often generate generic, bloated, or overly obvious comments (e.g., `// This function adds a and b`) while ignoring the underlying architecture, edge cases, or version control headers. They also frequently forget to update `README.md` files or add JSDoc/Python docstrings.

## The Solution (This Skill)
By injecting this skill into your AI Agent, it learns the **Enterprise Documentation Standard**:
1. **Comprehensive Docstrings**: It enforces standard docstring formats (JSDoc, PEP 257) with proper types, return values, and edge-case behaviors.
2. **Version Control Headers**: It automatically injects copyright headers, author attributions, and timestamps when creating new files.
3. **Clean Code Explanation**: It focuses on documenting *why* code exists and its *business logic*, rather than just rephrasing *what* the syntax does.
4. **README Generation**: It builds structured, highly readable READMEs that include architecture overviews, setup instructions, and dependency lists.
